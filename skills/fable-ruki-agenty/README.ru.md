# Фейбл руки-агенты

Ручной режим оркестрации для Claude Fable 5 в условиях урезанных лимитов: Fable — только голова. Он думает, резолвит развилки и пишет полные самодостаточные спеки в тела GitHub issues; всю механику — кодинг, ревью, gh-операции, чтение, разведку, DoD-верификацию — делают дешёвые руки-агенты.

Рука — это пара «харнесс + модель», и конвейер от неё не зависит: он знает роли (скаут, исполнитель, gh-рука, верификатор, ревьюер) и четыре операции (диспатч, продолжить тред, свежий контекст, отчёт). Все привязки к конкретным тулам и моделям живут в одном файле — [`references/channels.md`](references/channels.md); по умолчанию руки — субагенты Claude Code, и смена окружения или руки — правка этого файла, а не конвейера.

## Ключевые правила

- **Гейт деления.** Делить работу между агентами нельзя, пока выполнение одной части может изменить постановку другой части. Три маркера «рано делить»: бриф ветвится по неоткрытым фактам («если окажется, что…»), агенты перебивали бы друг друга, сидя в одной комнате, или их отчёты придётся стыковать, а не складывать. Первые два лечатся только фазово — копает одна голова; третий — предварительной фиксацией связующих решений в контракт.
- **Петля насыщения для исследования и диагностики.** Когда список объектов разведки сам является предметом разведки (рынок и конкуренты, отладка бага, разбор инцидента), один скаут работает в одном длинном контексте, пока карта не перестанет расти — механическое стоп-правило: пять поисков подряд без новых сущностей. Только родившийся закрытый список раздаётся веером параллельных скаутов: каждому вкладывается вся карта, и каждый обязан вернуть секцию «новое, чего нет на карте». Дальше интеграция и, если карта выросла, ещё круг.
- **Issue-first, без исключений.** Любая задача — из бэклога или из чата — сначала становится issue, тело которого — полная спека (цель, контекст, решённые развилки, шаги, границы, DoD). Только потом диспатч.
- **Диспатч указателем.** Исполнитель читает тело issue через `gh issue view N`; промпт — только операционный конверт.
- **Отдельный верификатор на каждую задачу.** Рука с чистым контекстом прогоняет проверку из DoD; принимает не тот, кто делал. Вердикт «непроверяемо» легален и не закрывает issue.
- **Лестница при провале:** две доработки тем же исполнителем → свежий исполнитель с диагнозом верификатора → лейбл `blocked` и короткий доклад пользователю.
- **Спекование вперёд.** Пока исполнитель работает, Fable пишет спеки следующих задач, а не ждёт.
- Параллельность определяется сначала гейтом деления, затем пересечением файлов — и никогда количеством агентов: задачи по одному файлу — последовательно с прямыми коммитами в main; файлово-независимые группы с уже замороженными брифами — параллельно в ворктри.

## Зачем

Токены Fable — самый дорогой ресурс сессии. Каждый tool call, который могла сделать модель дешевле, — сожжённый лимит. Скилл превращает Fable в чистого спекописателя и диспетчера — но суждение не делегируется никому, как и фаза, вопросы которой ещё открыты.

Скилл вызывается вручную (`disable-model-invocation: true`), моделью автоматически не подтягивается.
