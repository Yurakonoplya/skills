# Внешние каналы — механика

Дефолт конвейера — штатные субагенты (§ Субагенты и каналы SKILL.md); этот файл нужен, только если на старте конвейера пользователь подключил внешний канал. Здесь всё, что привязано к конкретным продуктам, — текст SKILL.md от канала не зависит.

## Codex (OpenAI)

Роли: **ревьюер финального ревью** — главная ценность, другое семейство моделей снимает модельно-коррелированные слепые зоны; **исполнитель** — по решению пользователя. Бонус канала: жжёт подписку OpenAI, а не лимиты Anthropic.

### Подключение (проверено 09.07.2026)

Бинарник поставляется десктопным приложением Codex и в PATH не добавлен — команда `codex` из терминала не находится, хотя всё установлено. Путь не хардкодить (хэш в нём меняется при обновлениях), а брать из конфига:

```bash
CODEX="$(sed -n "s/^CODEX_CLI_PATH = '\(.*\)'$/\1/p" "$HOME/.codex/config.toml" | head -1)"
# fallback: CODEX="$(ls "$HOME/AppData/Local/OpenAI/Codex/bin"/*/codex.exe | tail -1)"
"$CODEX" --version   # прекондишн: версия печатается → канал готов
```

Авторизация и модель (gpt-5.5, effort high) — из `~/.codex/config.toml`, флагами не дублировать. Версия не напечаталась — пользователю одна строка, роли остаются субагентам.

### Операции (CLI, headless)

Все вызовы делает Bash-субагент (запрет № 1), из рабочей директории задачи (`cd <path>` перед вызовом).

- **Диспатч:** `"$CODEX" exec --skip-git-repo-check --sandbox workspace-write "<конверт из § 3>"`. Проверено: файлы пишет, в stdout возвращает диффы, финальный ответ и расход токенов — субагент пересказывает их в отчёт по скретчпад-протоколу.
- **Продолжить тред:** `"$CODEX" exec resume <session-id>` (или `--last`, если сессия Codex последняя). **Свежий контекст** — новый `exec`.
- **Финальное ревью:** `"$CODEX" exec review` — нативный ревьюер репозитория. Оси проверки обязательны (§ 6 SKILL.md); как их передать — проверить флаги при первом запуске, результат в журнал сбоев.

### Боевые ограничения

- **Сеть внутри сандбокса Codex не гарантирована** (инцидент 04.07.2026 у автора исходного скилла): gh-операции остаются gh-агенту; Codex — только файлы и коммиты.
- **Пустой или оборванный промпт при старте** — не респавнить: продолжить тред и переслать текст задачи (правило § 3 SKILL.md).
