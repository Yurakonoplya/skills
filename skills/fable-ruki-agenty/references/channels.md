# Каналы рук — механика

Читать перед первым диспатчем канала в сессии. Конвейер (SKILL.md) знает только четыре операции: диспатч, продолжить тред, свежий контекст, отчёт. Здесь — как эти операции выглядят в каждом канале.

Рука — пара «харнесс + модель». Харнесс определяет механику вызова, модель задаётся флагом. Смена модели (в том числе на другую линейку) — правка флага в этом файле и строки в таблице SKILL.md, не переписывание конвейера.

## Codex — дефолтный канал

Кодинг, ревью, gh-операции, CLI-механика, DoD-верификация, укладка файлов. Плагин `codex`, runtime codex-companion. Модель — **`--model gpt-5.5`, всегда явным флагом**, не из дефолта `~/.codex/config.toml`.

Сеть в сандбоксе Codex закрыта (инцидент 04.07.2026): gh-операции выполняет отдельная рука своим Bash, Codex — только файлы и коммиты.

### Прекондишн перед первым Codex-диспатчем конвейера

Прогоняет gh-рука своим Bash, не Fable (запрет № 1 SKILL.md); шаг входит в чек-лист § Старт конвейера.

```bash
CODEX_PLUGIN="$(printf '%s\n' "$HOME"/.claude/plugins/cache/openai-codex/codex/*/ | sort -V | tail -1)"
node "${CODEX_PLUGIN%/}/scripts/codex-companion.mjs" setup --json   # ждать ready:true и auth.loggedIn:true
```

Не ready — стоп конвейера, пользователю одной строкой: нужен `/codex:setup` (установка и логин — его решение).

### Кодинг-задача

Диспатч через Agent tool: `subagent_type: "codex:codex-rescue"` (форвардер тонкий, мозги — Codex). Текст задачи = маршрутные флаги + конверт из § Диспатч.

- `--wait --write --model gpt-5.5` — всегда. Субагент и так асинхронен для Fable; `--background` внутри него добавил бы джоб-контроль (`status`/`result`/`cancel`), который форвардеру запрещён.
- `--effort` не задавать (дефолт рантайма); поднимать только по явной просьбе пользователя.
- Продолжить тред (доработка после верификатора) — новый вызов `codex:codex-rescue` с `--resume` в тексте: тот же Codex-тред, контекст правок сохранён.
- Свежий контекст (ступень 3 лестницы приёмки) — `--fresh`.
- Ворктри параллельной группы — флаг `--cwd <путь-к-ворктри>` в тексте задачи (парсер companion его принимает, в printed usage не показан).

### Ревью

Нативный ревьюер Codex, не `task`. Запускает gh-рука одним Bash-вызовом и возвращает stdout как есть:

```bash
node "${CODEX_PLUGIN%/}/scripts/codex-companion.mjs" adversarial-review --wait \
  --base <стартовый SHA конвейера> --scope branch --model gpt-5.5 "<оси проверки от Fable>"
```

Вердикты триажит Fable. Оси проверки нужны всегда (это спека ревью); `review` без focus-текста — деградация, для финального ревью не использовать.

Если исполнителем был Codex, финальное adversarial-ревью лучше гнать вторым каналом: одна модель систематически пропускает одни и те же классы ошибок (§ 6 SKILL.md).

### Боевые ограничения (03.07.2026)

- **Пустой промпт при спавне.** `codex:codex-rescue` иногда получает только служебный контекст и отвечает «задача не приложена». Не респавнить: переслать текст задачи тому же агенту через `SendMessage` — со второго раза доезжает.
- **`--cwd` в ворктри не обкатан.** При первом параллельном прогоне проверить, что дифф лёг в ворктри, а не в основную копию; до проверки Codex-исполнителей диспатчить последовательно. Правило пересечения файлов остаётся в силе.

## Orca — опциональный канал

Видимые терминалы с внешними агентами. Роли по умолчанию: разведка, чтение, скауты. Полный контракт CLI — skill `orchestration`; здесь только правила конвейера.

**Когда Orca, а не Codex:** (а) разведка, чтение или скаут; (б) рука — внешний агент в терминале; (в) пользователь хочет видеть работу вживую (стрим). Дефолт кодинга остаётся Codex.

**Как** (orca CLI гоняет gh-рука, Bash у Fable не появляется):

```
terminal create --command "<agent-cli>" --title "worker: <задача>"
terminal wait --for tui-idle
task-create --spec "<конверт-указатель: gh issue view N + абсолютные пути>"
dispatch --task … --to <term> --inject
check --wait --types worker_done,escalation,decision_gate
```

Правила:

- **Терминалы пользователя не переиспользовать.** Под руки оркестратор открывает новые — `terminal create`, титул с префиксом `worker:`. Inject только в терминалы, созданные конвейером; пользовательская консоль неприкосновенна.
- **Абсолютные пути в спеке всегда, поиск по ФС руке не давать.** Cwd терминала ненадёжен (боевой случай: терминал числился в одном репо, orchestration вёл его в ворктри другого), а задача «найди файл» вешает агента намертво (боевой случай 03.07.2026: спиннер 10m+ на глобальном Glob-поиске). В спеке — точные пути файлов.
- **Контекст — только текстом.** Истории сессий Orca не переносит; всё, что рука должна знать, Fable пишет в `--spec`/`send --body` дословно. Уточнение работающей руке — `send --body`; полный handoff — `terminal send --text --enter`.
- **`worker_done` ≠ приёмка.** Вердикт руки не заменяет верификатора; лестница провалов действует.
- **Продолжить тред** — `send --body` в тот же терминал (контекст жив). **Свежий контекст** — новый терминал + новая задача.
- **Эскалации** (`escalation`/`decision_gate`) резолвит Fable как развилки: решение + строка обоснования, ответ через orca-руку.
- **Таймаут `check --wait` без события ≠ провал.** Рука делает `terminal read` и возвращает факт: работает / застрял / умер.
